Detailed Description

Defines used to configure the baud rate of the UART.

If UART_RATE_DEFAULT is used as the baud_rate_enum parameter, then the rate and UART config are defined by the current CS settings.

Otherwise, baud_rate_enum is selected from the values below, and config is a bitfield described by UART configuration. Non-standard baud rates can be configured; contact CSR for more information.

  • UART_RATE_2K4 = 0x000a
  • UART_RATE_9K6 = 0x0028
  • UART_RATE_19K2 = 0x004e
  • UART_RATE_38K4 = 0x009e
  • UART_RATE_57K6 = 0x00eb
  • UART_RATE_115K2 = 0x01d9
  • UART_RATE_230K4 = 0x03af
  • UART_RATE_460K8 = 0x0760
  • UART_RATE_921K6 = 0x0ebf
  • UART_RATE_1382K4 = 0x161f
  • UART_RATE_1843K2 = 0x1d7e
  • UART_RATE_2764K8 = 0x2c3d
  • UART_RATE_3686K4 = 0x3afc
  • UART_SLOW_RATE_2K4 = 0x12c0

Baud rates defined as UART_SLOW_RATE_X indicate rates that drive the UART from the 32kHz clock rather than the 16MHz crystal. This has the advantage that the UART can be used when Deep Sleep is enabled without having to wake the chip via the WAKE pin before transmitting to it. However, the disadvantage is that only a limited set of low baud rates are available in this mode. If a higher baud rate is required then either Deep Sleep cannot be used or the Host processor must assert WAKE before transmitting.
