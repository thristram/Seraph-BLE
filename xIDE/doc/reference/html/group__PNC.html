Detailed Description

Incorrect function from firmware should normally trigger a fault. Based on CSKey settings this can trigger a watchdog.

Function Documentation

void Panic ( uint16  panic_code )

Raise a system panic, normally resetting the chip.

In normal operation, this function will not return since it resets the chip. If the CS Key err_panic ("Cause faults to panic") is changed from its default setting to prevent FAULT_APPLICATION_PANIC from panicking the chip, this function will return. This is not normally advisable.

Parameters:
panic_codeAn application-defined value. This value is preserved across resets to assist debugging. Applications are strongly advised to refrain from using 0 as valid panic code, even though it is currently treated as valid code by firmware. See PANIC_NONE.
Returns:
Nothing. Does not return under normal circumstances.

Allows the user application to retrieve last FW fault ID.

FW fault ID is stored in persistent memoory. This API does not take care of persistent memory validity. This could return FAULT_NONE in case FW determines that whole persistent memory needs to be reset at boot up. See fault.xml for more information on fault ID.
