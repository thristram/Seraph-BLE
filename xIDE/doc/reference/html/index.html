CSR µEnergy® Firmware Library Documentation

Introduction

This set of help pages documents the API's available to application writers when linking against the CSR µEnergy® Firmware library.

Chip Support

The majority of the APIs described in this documentation are supported by all chips in the CSR µEnergy® product range (CSR1000™, CSR1001™, CSR1010™ and CSR1011™). In cases where there are differences in support then this will be stated for each individual API.

It is important that when building a project to select the correct chip. To select the desired chip, open the project in xIDE. Then go to the Project menu and select Properties. Select Configuration Properties...Build System...General in the left pane then look for "Target HW" in the right pane. The options include each supported chip family (e.g. CSR101x A05 for CSR1010™ & CSR1011™) plus an additional option "Auto Detect". Auto Detect attempts to detect the chip of an attached development board - this will only work if a device is available.

Broadly the differences between the chips can be described as:

  • CSR1000™ & CSR1001™
    • The only difference between these two chips is the additional PIO pins available on the CSR1001™ (32 PIOs versus 11 PIOs on CSR1000™).
    • Both chips support the same set of APIs. Accessing unavailable PIOs on the CSR1000™ is treated as a null operation (no error will be returned).
  • CSR1010™ & CSR1011™
    • As with CSR1000™ & CSR1001™, CSR1010™ has 11 PIOs and CSR1011™ has 32 PIOs. APIs that access these PIOs are identical between the two chips, and accessing unavailable PIOs on the CSR1010™ does not result in an error.
    • The CSR101x chips also include two hardware quadrature decoders. There are new APIs to configure and use these decoders that are not available on the CSR100x chips.
    • The CSR101x chips contain substantially more of the CSR µEnergy® Firmware library in ROM. This has freed up a large amount of extra RAM for application use (code and/or data).
